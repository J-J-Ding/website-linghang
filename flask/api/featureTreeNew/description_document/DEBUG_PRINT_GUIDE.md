## 打印符号说明

| 符号 | 含义 |
|------|------|
| 🚀 | 开始执行 |
| 📌 | 步骤标记 |
| ✅ | 成功完成 |
| ❌ | 失败/错误 |
| ⚠️ | 警告信息 |

## 调试技巧

### 1. 快速定位问题
通过打印的 Step 编号，可以快速定位图谱生成在哪个阶段出现问题。

### 2. 检查数据流
- **Step 1** 确认种子组件是否正确查询到
- **Step 3** 确认组件树节点是否加载
- **Step 4** 确认模块关系是否正确提取
- **Step 5-7** 确认 iCenter 页面解析是否成功

### 3. 验证节点数量
通过最终的节点类型分布，可以验证：
- 是否有预期类型的节点
- 节点数量是否合理
- 是否有遗漏的节点类型

### 4. 性能分析
通过打印的时间戳（如果需要，可以在关键步骤添加时间戳），可以分析：
- 哪个步骤耗时最长
- 是否需要优化查询逻辑
- 是否需要添加缓存

## 日志级别说明

| 打印类型 | 说明 | 是否输出到控制台 |
|----------|------|------------------|
| `print()` | 详细调试信息 | ✅ 是 |
| `logger.info()` | 一般信息 | ✅ 是（配置决定） |
| `logger.warning()` | 警告信息 | ✅ 是 |
| `logger.error()` | 错误信息 | ✅ 是 |

## 关闭打印信息

如果生产环境不需要详细的打印信息，可以通过以下方式关闭：

1. **修改代码**：将 `print()` 语句改为 `logger.debug()`
2. **配置日志级别**：设置日志级别为 WARNING 或 ERROR
3. **环境变量控制**：添加环境变量 `DEBUG_PRINT=0` 来控制
