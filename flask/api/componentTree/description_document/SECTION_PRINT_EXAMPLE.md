# 章节提取打印信息示例

## 📋 修改说明

在 `graph_service.py` 的 Step 5 中，增加了详细的章节标题和内容对应关系打印信息，使用指定的隔离标记格式。

```

   🔍 开始提取章节...
   📊 提取结果：3 个章节
   ✅ 提取到 3 个章节:

   [==================================================]
   章节 1/3
   标题：5.3.1 流程设计
   类型：5.3.1
   字数：1250
   <====Context begin=======================================================>
   页面状态初始*必填*，选择状态：初始、已初审、修订中、修订中、已定稿页面守护人*必填*...
   （显示前 500 字符，超过则显示省略号）
   <====Context end=========================================================>

   [==================================================]
   章节 2/3
   标题：5.7.7 FT 代码文件设计
   类型：5.7.7

   <====Context begin=======================================================>

   [==================================================]
   章节 3/3

   类型：6
   字数：2100
   <====Context begin=======================================================>
   描述组件的使用说明及使用约束，特别指出：描述时需要基于场景示例...
   <====Context end=========================================================>

   ✅ 添加了 3 个章节节点和关系边
```

## 📊 打印信息包含的字段

每个章节都会打印以下信息：

1. **分隔线**: `[==================================================]` (50 个等号)
2. **章节序号**: `章节 1/3` (当前章节/总章节数)
3. **标题**: 完整的章节标题
4. **类型**: 章节类型（如 5.3.1、5.7.7 等）
5. **字数**: 章节内容的字符数
6. **开始标记**: `<====Context begin=======================================================>`
7. **内容预览**: 前 500 字符（超过则显示省略号）
8. **结束标记**: `<====Context end=========================================================>`

## 🔍 隔离标记格式

严格按照要求的格式：

```
<====Context begin ==================================================>
<====Context   end ==================================================>
```

实际打印时会根据等号数量自动对齐：
- `begin` 后面跟 55 个 `=`
- `end` 后面跟 57 个 `=`
- 保持整体宽度一致（73 字符）

## 💡 使用场景

### 1. 开发调试
```bash
# 启动 Flask 应用查看日志
cd flask/api
gunicorn -w 4 -b 0.0.0.0:3001 --timeout 360 --access-logfile - --error-logfile - app:app
```

### 2. 验证章节内容
通过打印信息可以快速确认：
- 章节标题是否正确提取
- 章节内容是否完整
- 章节标题和内容的对应关系是否正确
- 内容字数是否符合预期

### 3. 问题排查
如果章节内容有问题，可以通过打印信息快速定位：
- 内容为空：检查 iCenter 页面解析逻辑
- 内容截断：检查数据库字段类型（已改为 TEXT）
- 标题错误：检查章节提取逻辑
